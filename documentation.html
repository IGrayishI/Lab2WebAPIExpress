Book API Documentation

1. Get All Books

Endpoint: [GET] /book

Description: Get a list of all books.

Example Response:


[
    {
        "id": 1,
        "title": "The Great Gatsby",
        "author": "F. Scott Fitzgerald"
    },
    {
        "id": 2,
        "title": "To Kill a Mockingbird",
        "author": "Harper Lee"
    },
    // ... more books
        
]

2. Get Book by Id

Endpoint: [GET] /book/{id}

Description: Get a single book.

Request Parameters:

id: The ID of the book to be gotten.

Example Response:


    {
        "id": 1,
        "title": "The Great Gatsby",
        "author": "F. Scott Fitzgerald"
    }

3. Create a New Book

Endpoint: [POST] /book

Description: Create a new book by providing the book data in the request body.

Request Body:


    {
        "title": "The Catcher in the Rye",
        "author": "J.D. Salinger"
    }

Example Response:


    {
        "id": 3,
        "title": "The Catcher in the Rye",
        "author": "J.D. Salinger"
    }

4. Update Book Information

Endpoint: [PUT] /book/{id}

Description: Update information for a specific book identified by its ID.

Request Parameters:

id: The ID of the book to be updated.

Request Body:


    {
        "title": "New Title",
        "author": "New Author"
    }

Example Response:


    {
        "title": "New Title",
        "author": "New Author"
    }

5. Delete Book

Endpoint: [DELETE] /book/{id}

Description: Deletes a single book.

Request Parameters:

id: The ID of the book to be deleted.

Note:

  • >All endpoints may return an error status (e.g., 500 Server Error) if there are issues with the API or data processing.
  • Ensure proper error handling on the client side to handle these cases.
